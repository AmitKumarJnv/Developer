# Backend Developer for Data Science – Learning Roadmap

This repository organizes my learning journey to become a backend developer specialized in data science.  
Each folder contains code examples, notes, and small projects for a specific technology.

## 📂 Folder Structure

| Folder | Description |
|--------|-------------|
| `01-markdown` | Markdown syntax, documentation best practices, and writing technical docs. |
| `02-python` | Python core (data structures, OOP, typing, decorators, async, testing with pytest). |
| `03-dsa` | Data Structures & Algorithms (arrays, trees, graphs, recursion, Big O, problem-solving patterns). |
| `04-fastapi` | FastAPI framework: routes, dependency injection, background tasks, OpenAPI, and API testing (`httpx`, `pytest-asyncio`, mocking). |
| `05-databases` | SQL (PostgreSQL), NoSQL (MongoDB, Redis), ORMs (SQLAlchemy), migrations. |
| `06-docker` | Dockerfiles, image layering, volumes, networking, docker-compose for multi‑service apps. |
| `07-kubernetes` | Pods, deployments, services, ConfigMaps, Ingress, Helm charts, Minikube. |
| `08-projects` | End‑to‑end projects combining all the above (e.g., real‑time ML inference API). |

## 🚀 How to Use

1. Navigate to any folder.
2. Each folder has its own `requirements.txt` instructions in its own `README.md`.
3. Use separate virtual environments per chapter to avoid dependency conflicts (pattern: `.venv_chap_*` – already in `.gitignore`).
